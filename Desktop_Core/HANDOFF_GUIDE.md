# 🚀 StockScan POS - دليل تسليم المشروع للعميل

تهانينا على الوصول لهذه المرحلة! هذا الدليل سيشرح لك بالتفصيل كيفية تحويل الكود إلى برنامج تثبيت احترافي (Installer) وتسليمه لعميلك ليعمل على جهازه بضغطة واحدة.

> [!IMPORTANT]
> **لقد قمت بتحويل البرنامج ليعمل بشكل مستقل تماماً (Standalone).**
> العميل **لن يحتاج** لتثبيت Node.js أو أي برامج إضافية. البرنامج يحتوي على كل ما يلزم داخله.

---

## 🛠️ الخطوة 1: تجهيز النسخة النهائية (Build)

أولاً، نحتاج للتأكد من أن جميع الأكواد جاهزة ومجمعة. افتح الـ Terminal في المجلد الرئيسي للمشروع ونفذ الأوامر التالية بالترتيب:

1.  **بناء واجهة المستخدم (Front-end Build):**
    ```bash
    npm run build:frontend
    ```
    *(لقد قمت بهذه الخطوة بالفعل بنجاح، لا داعي لتكرارها إذا لم تغير الكود)*

2.  **تثبيت المكتبات اللازمة للتغليف (Dependencies Check):**
    ```bash
    npm install
    ```

---

## 📦 الخطوة 2: إنشاء ملف التثبيت (.exe)

بعد انتهاء الأمر السابق، سنقوم الآن بإنشاء ملف الـ Setup النهائي:

1.  **تشغيل أمر التغليف:**
    ```bash
    npm run dist
    ```
2.  انتظر قليلاً حتى ينتهي المعالج (قد يستغرق 2-5 دقائق).

---

## 📂 الخطوة 3: تحديد الملف المطلوب تسليمه

بمجرد انتهاء الأمر، سيظهر لك مجلد جديد باسم `dist_electron`. ادخل داخله وستجد ملفاً باسم:
**`StockScan Setup 1.0.0.exe`**

**هذا هو الملف الوحيد الذي ستحتاج لإعطائه للعميل.** يمكنك رفعه على Google Drive أو وضعه على "فلاشة" وتسليمه له.

---

## 💻 الخطوة 4: التثبيت على جهاز العميل

عندما يفتح العميل الملف على جهازه، ستحدث الخطوات التالية تلقائياً:
1.  سيظهر شريط تحميل (جاري التثبيت).
2.  سيقوم البرنامج بإنشاء أيقونة على سطح المكتب باسم **StockScan**.
3.  بمجرد فتح البرنامج، سيقوم بتشغيل "قاعدة البيانات" و "السيرفر" داخلياً دون أن يلاحظ العميل أي أكواد برمجية.

---

## 🔍 ملاحظات هامة لك:
*   **مكان حفظ البيانات:** تم ضبط البرنامج ليحفظ الداتا في مجلد `AppData` الخاص بويندوز على جهاز العميل. هذا يضمن أن البيانات لن تضيع حتى لو قام العميل بتحديث البرنامج لاحقاً.
*   **بدون إنترنت:** البرنامج يعمل بالكامل Offline، ولن يحتاج العميل لاتصال بالإنترنت أبداً.
