# دليل تشغيل وتسليم مشروع StockScan POS 🚀

هذا الملف يحتوي على كافة التفاصيل التقنية والعملية لتشغيل المشروع، إدارته، وتسليمه للعميل بشكل احترافي.

---

## 1. فكرة المشروع وآلية العمل (Architecture)
مشروع **StockScan** هو نظام إدارة مخازن ونقاط بيع (POS) مصمم ليعل بشكل **أوفلاين تماماً (Offline)** دون الحاجة لإنترنت.

### المكونات التقنية:
- **الواجهة (Frontend):** مبنية بـ React و Vite لتوفير تجربة مستخدم سريعة وسلسة.
- **الخلفية (Backend):** سيرفر محلي يعمل بـ Node.js و Express.
- **قاعدة البيانات (Database):** يستخدم **SQLite** (ملف محلي باسم `stockscan.db`) لضمان بقاء البيانات على جهاز العميل.
- **محرك التشغيل (Desktop Wrapper):** يستخدم **Electron** لدمج السيرفر والواجهة في تطبيق ويندوز (.exe) واحد.

### كيف يعمل البرنامج؟
عند تشغيل التطبيق، يقوم Electron بتشغيل "سيرفر خلفي" (Background Process) بشكل مخفي، ثم يقوم بفتح واجهة المستخدم وتوجيهها لهذا السيرفر. يتم تخزين واستدعاء كافة البيانات من ملف قاعدة البيانات الموجود في مخزن بيانات التطبيق (AppData).

---

## 2. كيفية التشغيل (للمطور)
لتشغيل المشروع في وضع التطوير (Development Mode):
1. افتح مجلد المشروع في التيرمنال.
2. قم بتثبيت المكتبات (إذا لم تكن مثبتة): `npm install`
3. شغل الأمر التالي:
   ```bash
   npm run dev
   ```
سيقوم هذا الأمر بتشغيل السيرفر، الواجهة، ونافذة Electron في وقت واحد.

---

## 3. كيفية تسليم المشروع للعميل (Build & Packaging)
لتحويل الكود إلى تطبيق احترافي (Installer) للعميل:

1. **بناء الواجهة:**
   ```bash
   npm run build
   ```
2. **إنشاء ملف التثبيت (.exe):**
   ```bash
   npm run dist
   ```
3. **أين تجد الملف؟**
   بعد اكتمال العملية، اذهب إلى مجلد `dist_electron`. ستجد ملفاً باسم `StockScan Setup 1.0.0.exe`. هذا هو الملف الوحيد الذي يحتاجه العميل لتثبيت البرنامج على جهازه.

---

## 4. كيفية الوصول لقاعدة البيانات عند العميل
إذا طلب العميل تعديل بيانات يدوياً أو أردت سحب نسخة احتياطية من البيانات من جهازه المحلي لجهازك:

1. **مكان قاعدة البيانات على جهاز العميل:**
   اذهب للمسار التالي (استبدل `YourName` باسم مستخدم ويندوز):
   `C:\Users\YourName\AppData\Roaming\StockScan\data\stockscan.db`
   *(أو اكتب `%APPDATA%\StockScan\data` في شريط مسار المجلدات وسيفتح لك مباشرة).*

2. **كيف تفتحها على جهازك الشخصي؟**
   - خد نسخة من ملف `stockscan.db` من عند العميل.
   - استخدم برنامج [DB Browser for SQLite](https://sqlitebrowser.org/).
   - افتح الملف بالبرنامج، وستتمكن من رؤية الجداول (المستخدمين، المنتجات، الفواتير) وتعديلها أو تصديرها لملف Excel.

---

## 5. الحسابات الافتراضية (Default Credentials)
يأتي البرنامج بحسابات افتراضية مبرمجة مسبقاً (Seed Data) لتسهيل الدخول لأول مرة:

| الحساب | البريد الإلكتروني (Email) | كلمة السر (Password) | الصلاحيات |
| :--- | :--- | :--- | :--- |
| **المدير** | `admin@stockscan.com` | `admin123` | كامل الصلاحيات (التقارير، المخزون، المستخدمين) |
| **الكاشير** | `cashier@stockscan.com` | `cashier123` | البيع وإصدار الفواتير فقط |

---

## 6. تفاصيل المخزون (Inventory Management)
- **إضافة المنتجات:** يتم تخزين الباركود، اسم المنتج، الكمية، سعر التكلفة، وسعر البيع.
- **الخصم التلقائي:** عند إتمام أي فاتورة بيع، يقوم البرنامج تلقائياً بخصم الكميات المباعة من المخزون في قاعدة البيانات.
- **تنبيهات المخزون:** يمكنك مراقبة الكميات من صفحة "المخزون" في البرنامج، وإذا وصلت الكمية لصفر لن يتمكن الكاشير من بيع المنتج إلا بعد إضافة كمية جديدة.
